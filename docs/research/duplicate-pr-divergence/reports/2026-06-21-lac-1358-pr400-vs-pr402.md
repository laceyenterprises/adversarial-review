## Final Remediation On Survivor

The final rescue pass on #400 addressed every finding from the 2026-06-21 `Comment only` review:

1. Security-critical parser and allowlist duplication.
   - Added `src/ama/ham-provenance.mjs`.
   - Moved `parseCommitTrailers`, `parseRemediatedFindingsTrailer`, `hamAuditCommentAuthorMatches`, and `HAM_AUDIT_COMMENT_AUTHOR_LOGINS` into one shared module.
   - Tightened trailer parsing to stop at the first blank above the contiguous trailer block.
2. HAM rejection reason discarded on requeue.
   - The fast-merge path now appends `ham-eval: <reason>` to the requeue reason and logs the exact HAM authorization rejection.
   - Operators can distinguish missing HQ roots, missing audit files, invalid provenance, missing comments, untrusted audit files, and closer-lease drift.
3. Fast-merge lane used `process.env` instead of threaded runtime env.
   - `pollFastMergeQueue` now accepts `env` and passes it to `processFastMergePR`.
   - `runFastMergeClosePathIsolated` threads its runtime env into the fast-merge close path.
4. Audit JSON authority needed a local trust check.
   - The fast-merge HAM predicate now stats the keyed AMA audit JSON, requires it to be a regular file, owned by the configured HQ owner, and not world-writable.
   - Missing or untrusted files fail closed to normal review.
5. `in_progress` audit records needed a race/integrity story.
   - A `succeeded` audit remains terminal authority.
   - An `in_progress` audit authorizes the changed-head carve-out only while the matching AMA closer lease for `(repo, pr, head)` is `dispatched` with an LRQ.
   - The governing specs now document this distinction.

## Research Notes

This pair is a compact example of how a duplicate retry can look cleaner while being less safe. #402 optimizes for a local syntactic proof: parent commit, non-empty diff, trailer strings, and a PR comment. #400 optimizes for authority provenance: the local AMA audit record, a trusted file boundary, a matching closer lease, GitHub corroboration, and fail-closed requeue behavior.

The durable lesson is that merge-authority exceptions need an authority anchor outside the branch author. Commit trailers and PR comments are good breadcrumbs, but they are not a trust root. The stronger branch is the one that makes the unreviewed-head exception boringly auditable and hard to forge, even if that costs more code and tests.
