## Decision

#382 is the stronger survivor. It makes the merged `build_completions` signal
the authoritative closer-completion predicate, updates the AMA SPEC/runbook
surfaces, adds a session-ledger read adapter, and covers the producer-read paths
in tests. #381 is useful and smaller, but it still treats HQ terminal status and
AMA audit outcomes as the main ownership evidence and leaves the completion
source less explicit.

#381 should close unmerged after #382 lands. Its useful ideas were folded into
#382: active HQ status must retain ownership even when a stale failed audit is
present, terminal-success without proof remains bounded rather than blind, and
operators need clear runbook language for producer-lag behavior.

## Why #381 Was Not The Survivor

#381 fixes the original blind relaunch class with a compact change: it stops
force-deleting terminal leases, adds retry exhaustion around status probing, and
holds briefly on `succeeded` without readable audit. That is a good defensive
patch. Its weakness is that it remains centered on HQ status and AMA audit
outcomes, which are observations from the closer lane itself. The review notes
the remaining ambiguity: a delayed audit can produce misleading durable outcome
state, carried-over `lastObservedStatus` can skip per-dispatch grace, and
`deferred` can later fall into merge-agent escalation by retry count.

Those are not merge blockers, but they show that #381 is still repairing symptoms
inside the closer lane rather than moving ownership to an external completion
fact.

## Why #382 Wins

#382 introduces the stronger boundary: a `build_completions` row with
`signal_kind='merged'` is the authoritative "this PR already completed" fact.
That moves the decision from closer self-observation to the Agent OS completion
ledger and makes `hq dispatch status=succeeded` plus AMA audit success
insufficient by themselves. The PR also updates the governing AMA SPEC, the
operator closure runbook, and the session-ledger read adapter tests, so the
operator contract moves with the code.

The final review correctly found that this broader design depends on an
out-of-repo producer. The hammer remediation on #382 therefore tightens the
contract in two ways:

- The merged-signal lookup is by `repo`/`pr_number`/`signal_kind`, not reviewed
  head SHA. Agent OS currently records the merge commit SHA in
  `build_completions.head_sha`, while AMA's `--match-head-commit` uses the PR
  head. The producer SHA is returned as evidence, not used as an equality gate.
- A clean missing row releases a terminal hold only when there is repo-level
  merged producer evidence. A schema-only rollout, missing table, unreadable
  ledger, or repo with no producer evidence retains the old hold semantics.

That absorbs the important safety pressure from #381 while preserving #382's
better source-of-truth model.

## Salvaged From #381

- Active HQ status retains ownership even if a stale terminal audit says failed.
- Terminal-success without authoritative merge proof stays bounded and
  diagnostic instead of blindly relaunching forever.
- Runbook language names producer-lag noise so operators do not misread bounded
  redispatch attempts as second merge attempts.
- Tests cover the active-status race and the stale terminal-success path.

## Final Remediation On Survivor

The #382 survivor now includes:

1. `readBuildCompletionProducerEvidence()` in the session-ledger adapter.
2. AMA closer lookup of merged rows without reviewed-head scoping.
3. Diagnostic exposure of the producer row's `head_sha` and whether it equals
   the reviewed head.
4. Fail-closed retention when the PR row is missing but repo-level merged
   producer evidence is absent.
5. Updated AMA SPEC/runbook text explaining the merge-commit-vs-PR-head SHA
   distinction.
6. This duplicate-divergence report and README corpus entry.

## Research Notes

SSG-06 is a useful duplicate because the smaller branch was locally cleaner but
the larger branch had the stronger authority boundary. #381 optimized for a
bounded retry repair inside the closer state machine. #382 optimized for a
completion-fact contract outside the closer state machine. Once the live
producer SHA mismatch was fixed, #382 became the better production artifact.

The lesson is not "choose the bigger PR." It is: for merge authority, prefer the
branch whose terminal decision rests on the most independent durable fact, then
force that branch to fail closed when the external producer contract cannot be
proven.
