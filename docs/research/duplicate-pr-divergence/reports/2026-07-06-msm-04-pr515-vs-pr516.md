direct single-route deletion, removes the standalone `ama-closer-prompt.md`
surface outright, and already had green Node 20 and Node 22 CI before rescue.

gaps: deleted incident-critical coverage, an early `daemon-clean-route`

silent stalls on temporary GitHub failures or a poison-pill loop on permanent
ones. The survivor therefore keeps #516's simpler route removal and folds only

## Salvage Folded Into #516

The survivor now carries these ideas from #515 or its review thread:

- Stale-head proof is required before an exhausted final hammer can resume on a
  live head that differs from the reviewed head.
- The dispatch record remains keyed by the stable reviewed head while the prompt
  targets the live PR head, preventing both re-hammer churn and destructive
  rollback to an old reviewed commit.
- Transient stale-head proof failures throw so the watcher tick retries later;
  permanent proof failures fail closed by declining stale-head hammer resume.
- `isHammerRemediableEligibilityMiss` rejects unproven `stale-review-head`
  before the exhausted-cycle auto-hammer waiver can fire.
- A clean eligible tick reconciles merged signals, active dispatches, and leases
  before returning `daemon-clean-route`.
- The survivor test suite regains focused coverage for stable-key/live-target
  wiring, stale-head proof polarity, active-dispatch reconciliation on clean
  routes, and the existing no-proof stale-head decline.

## Divergence Notes

The two harnesses optimized for different failure surfaces. The Codex branch
optimized for deleting the obsolete route cleanly: fewer files, current main,
green CI, and explicit deletion of the standalone prompt/template path. The
Claude Code branch optimized for operational aftermath: what happens if a
terminal hammer already pushed, the reviewed head is stale, and the system must
decide whether that stale live head is safe to merge. Both concerns were real.

The useful nondeterministic pattern is that the compact deletion branch produced
the better integration base, while the exploratory debug branch produced the
better partial-failure model. The merged survivor therefore should not be read
as "smaller won." It is "current, green, route-deleting base won, then absorbed
the losing branch's state-machine insight after correcting the error polarity."
